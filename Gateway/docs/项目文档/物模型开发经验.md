# ThingsKit 物模型开发经验总结

## 1. 物模型格式规范

### 1.1 ENUM类型必须使用specsList

```json
{
  "functionName": "节点角色",
  "identifier": "role",
  "functionType": "properties",
  "accessMode": "r",
  "specs": {
    "dataType": {
      "type": "ENUM",
      "specsList": [
        {"value": 0, "name": "root", "dataType": "ENUM"},
        {"value": 1, "name": "node", "dataType": "ENUM"}
      ]
    }
  }
}
```

**错误写法**（平台无法显示枚举值和描述）：
```json
"specs": {"0": "root", "1": "node"}
```

**正确写法**（平台显示枚举值和描述）：
```json
"specsList": [
  {"value": 0, "name": "root", "dataType": "ENUM"},
  {"value": 1, "name": "node", "dataType": "ENUM"}
]
```

**错误写法**（平台无法显示枚举值）：
```json
"specs": {"0": "root", "1": "node"}
```

### 1.2 BOOL类型必须有boolOpen/boolClose

```json
{
  "specs": {
    "dataType": {
      "type": "BOOL",
      "specs": {
        "boolClose": "离线",
        "boolOpen": "在线"
      }
    }
  }
}
```

### 1.3 STRUCT类型

**方式A：specs直接是子字段数组**
```json
{
  "specs": {
    "dataType": {
      "type": "STRUCT",
      "specs": [
        {"functionName": "父节点MAC", "identifier": "parent_mac", "dataType": {"type": "TEXT", "specs": {}}},
        {"functionName": "子节点数", "identifier": "child_count", "dataType": {"type": "INT", "specs": {"unit": "count"}}}
      ]
    }
  }
}
```

**方式B：使用specsList**
```json
{
  "specs": {
    "dataType": {
      "type": "STRUCT",
      "specsList": [
        {"identifier": "modbus_count", "name": "下挂设备数", "dataType": "INT", "specs": {"unit": "count"}},
        {"identifier": "collect_cycle", "name": "采集周期", "dataType": "INT", "specs": {"unit": "ms"}}
      ]
    }
  }
}
```

## 2. 物模型导入方式

### 2.1 手动导入（当前方式）
1. 进入产品详情
2. 点击"编辑物模型"
3. 点击"导入"按钮
4. 选择JSON文件

### 2.2 API导入（待研究）
- 通过API创建产品时，`profileData.thingsModel`字段可以设置物模型
- 但更新已有产品的物模型可能需要特定API

## 3. 本地文件格式 vs 平台格式

### 本地文件格式（things_model/*.json）
```json
{
  "properties": [
    {
      "functionName": "电压",
      "identifier": "voltage",
      "functionType": "properties",
      "accessMode": "r",
      "specs": {
        "dataType": {"type": "DOUBLE", "specs": {"unit": "V"}}
      }
    }
  ],
  "events": [...],
  "services": [...]
}
```

### 平台格式（API返回）
```json
{
  "functionType": "properties",
  "functionName": "电压",
  "identifier": "voltage",
  "callType": null,
  "accessMode": "r",
  "eventType": null,
  "functionJson": {
    "dataType": {"type": "DOUBLE", "specs": {"unit": "V"}}
  },
  "status": 1,
  "deviceProfileId": "xxx"
}
```

**关键区别**：本地用`specs`，平台用`functionJson`

## 4. DTU节点物模型设计

### 4.1 属性列表

| 标识符 | 名称 | 类型 | 说明 |
|--------|------|------|------|
| role | 节点角色 | ENUM | 0=root, 1=node |
| mac | MAC地址 | TEXT | |
| name | 设备名称 | TEXT | |
| online | 在线状态 | BOOL | |
| uptime | 运行时长 | INT | 单位：秒 |
| topology | 拓扑信息 | STRUCT | 包含parent_mac, child_count, child_macs |
| collect_config | 采集配置 | STRUCT | 包含modbus_count, collect_cycle, modbus_presets |

### 4.2 拓扑信息STRUCT

```json
{
  "functionName": "拓扑信息",
  "identifier": "topology",
  "specs": {
    "dataType": {
      "type": "STRUCT",
      "specs": [
        {"functionName": "父节点MAC", "identifier": "parent_mac", "dataType": {"type": "TEXT"}},
        {"functionName": "子节点数", "identifier": "child_count", "dataType": {"type": "INT"}},
        {"functionName": "子节点列表", "identifier": "child_macs", "dataType": {"type": "TEXT"}}
      ]
    }
  }
}
```

### 4.3 采集配置STRUCT

```json
{
  "functionName": "采集配置",
  "identifier": "collect_config",
  "specs": {
    "dataType": {
      "type": "STRUCT",
      "specs": [
        {"functionName": "Modbus设备数", "identifier": "modbus_count", "dataType": {"type": "INT"}},
        {"functionName": "采集周期", "identifier": "collect_cycle", "dataType": {"type": "INT"}},
        {
          "functionName": "Modbus预设",
          "identifier": "modbus_presets",
          "dataType": {
            "type": "STRUCT",
            "specsList": [
              {"identifier": "addr", "name": "设备地址", "dataType": "INT"},
              {"identifier": "dev_type", "name": "设备类型", "dataType": "ENUM", 
               "specs": {"0":"未定义","1":"三相电表","2":"单相电表","3":"温湿度变送器","4":"继电器","5":"其他"}}
            ]
          }
        }
      ]
    }
  }
}
```

### 4.4 设备类型枚举

| 值 | 名称 |
|----|------|
| 0 | 未定义 |
| 1 | 三相电表 |
| 2 | 单相电表 |
| 3 | 温湿度变送器 |
| 4 | 继电器 |
| 5 | 其他预留 |

### 4.1 BOOL类型
```json
{"online": true}   // 正确
{"online": 1}      // 也能工作
{"online": "true"} // 不推荐
```

### 4.2 ENUM类型
```json
{"role": "1"}      // 字符串
```

### 4.3 STRUCT类型
```json
{"topology": {"parent_mac": "...", "child_count": 1}}
```

### 4.4 MQTT Topic
```
网关属性: v1/devices/me/attributes
子设备遥测: v1/gateway/telemetry
```

## 5. 注意事项

1. 修改产品后需要手动导入JSON或重新创建产品
2. 设备重建后MQTT凭证会丢失，需要重新设置
3. ENUM类型必须有specsList才能正常显示枚举值
4. BOOL类型必须有boolOpen/boolClose才能显示文字
