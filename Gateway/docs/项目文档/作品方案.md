# 作品简介
**作品名称：基于星闪的无线电力透传系统**

本作品面向电力场景下传统 RS485 设备布线复杂、扩展困难、运维成本高、存量设备无线化改造不便等问题，设计并实现了一套基于星闪技术的无线电力透传系统。系统以星闪 DTU 为核心，支持三相电表、单相电表、温湿度变送器、继电器等 Modbus RTU设备接入，通过根节点、中继节点、叶子节点构建多级无线组网，实现现场设备数据的稳定采集、可靠转发、边缘分析和统一上云。

本系统突出“**易部署、强组网、低运维、可扩展**”的商用落地特性。在设备侧，DTU 支持蓝牙、扫码、电脑串口网页等多种便捷配置方式，用户仅需完成节点角色设定和参数下发，即可快速完成现场部署；在网络侧，系统充分利用星闪无线通信的低时延、抗干扰和跳频能力，解决传统无线 DTU 在密集部署场景下易串扰、覆盖受限、组网能力弱的问题；在平台侧，系统通过边缘网关实现多节点数据汇聚、压缩封装、统一上云、规则判断和本地联动控制，避免传统单台 4G DTU 分别联网带来的资费高、管理散、维护难等问题。作品可广泛应用于配电柜监测、电表集采、动力柜状态监测、工业能耗采集等场景，具备较强的工程应用价值和推广前景。

# 作品详细介绍
## 1. 项目定位：面向电力场景的无线化改造方案
本作品并非单纯的“无线串口替代器”，而是面向电力场景的**多设备接入、无线组网、边缘汇聚、统一上云**的一体化解决方案。  
系统重点服务于以下典型需求：

+  老旧配电柜、动力柜、表计箱等场景中布线难、改造成本高 
+  现场已部署大量 RS485/Modbus RTU 设备，但缺乏统一无线化接入手段 
+  传统单机 DTU 独立上云，存在流量成本高、管理分散、运维困难的问题 
+  电力现场对数据实时性、稳定性、抗干扰性和本地联动能力有较高要求 

因此，本作品的核心价值在于：**在不改变原有存量 485 设备的前提下，实现低成本、低侵入式的无线化和智能化升级。**

---

## 2. 解决的核心痛点
### （1）现场布线复杂，改造成本高
传统电力监测系统大量依赖有线部署，在配电室、动力柜、楼层配电箱、厂房支路等场景中，新增采集点往往意味着重新布线、开孔、走线、调试，施工量大、周期长、成本高。本系统通过星闪无线组网替代部分传统通信链路，显著降低部署难度。

### （2）传统 DTU 单点上云，资费和管理成本高
常见 4G DTU 多采用“**一台设备一张卡、一条链路独立上云**”模式。随着点位增多，不仅通信资费上升，还会导致设备管理分散、云端接入复杂、维护困难。本系统采用“**多节点采集 + 网关汇聚 + 统一上云**”架构，能够在保证现场接入能力的同时，有效降低整体联网和运维成本。

### （3）传统无线方案在密集场景下稳定性不足
在设备集中、金属柜体多、电磁环境复杂的电力场景中，传统短距或单跳无线方案常出现链路不稳、覆盖不足、串扰明显等问题。本系统基于星闪技术构建无线组网，增强了复杂场景下的数据可靠传输能力，更适合工业现场的部署需求。

### （4）现有星闪 DTU 多为单一星型接入，扩展能力有限
现有同类星闪接入方案较多采用“**一个主节点连接上位机/PLC，多个采集节点从属接入**”的单层模式，在覆盖范围、跨区域接入、节点扩展和复杂现场适配方面存在一定限制。本系统进一步实现了**根节点—中继节点—叶子节点**的多级组网结构，使得系统具备更强的网络延展性和场景适应性。

---

## 3. 系统总体架构
本作品采用“**设备侧 DTU + 星闪无线组网 + 边缘网关 + 云平台**”四层架构。

### （1）设备侧
各类 Modbus RTU 设备通过 RS485 接入 DTU，包括：

+  三相多功能电表 
+  单相电表 
+  温湿度变送器 
+  继电器模块 
+  其他兼容 Modbus RTU 的电力类仪表 

DTU 负责现场采集、过滤透传、控制和状态管理。

### （2）网络侧
多个 DTU 通过星闪构建无线网络，角色分为：

协调器，路由器，终端；

+ **根节点**：汇聚全网数据，与边缘网关通信 
+ **中继节点**：负责多跳转发，也可挂载本地设备 
+ **叶子节点**：负责末端设备采集与上报 

### （3）边缘侧
边缘网关接收根节点汇聚数据后，完成：

+  数据统一建模 
+  压缩封装 
+  本地缓存 
+  阈值判断 
+  告警生成 
+  联动控制 
+  统一上云 

### （4）平台侧
云平台实现：

+  设备状态监测 
+  历史趋势分析 
+  告警记录管理 
+  远程查看与管理 
+  后续 AI 趋势分析和异常识别扩展

## 4. 作品的核心功能设计
### （1）DTU 配置便捷，现场部署门槛低
本系统将“易部署”作为商用落地的重要设计目标，支持多种便捷配置方式：

+  手机 App 蓝牙连接配置 
+  扫码辅助连接与参数录入 
+  电脑通过 USB 转串口进行网页配置 

用户只需完成波特率、校验位、设备地址、网络角色等基础参数设置，即可完成节点初始化。  
同时，系统支持将某一节点配置为根节点，并录入其他节点的 MAC 信息，自动生成网络拓扑关系，降低现场部署难度，减少专业调试依赖。

### （2）支持多类型 RS485 设备统一接入
系统兼容多种 Modbus RTU 设备，不局限于单一表计。  
在电力场景中，既可以接入三相/单相电表进行计量采集，也可以接入继电器模块实现执行控制，接入温湿度变送器进行配电箱环境监测，满足“采集 + 状态 + 控制”的综合需求。

### （3）基于事件队列机制保证设备侧稳定运行
针对现场设备数量多、采集周期不同、命令控制与数据上报并发存在的情况，DTU 侧采用事件队列、任务调度、异常重试和缓存机制，保证采集、透传、控制过程有序执行，减少阻塞和丢包风险，提升长期运行稳定性。

### （4）实现多角色无线组网，扩展能力更强
区别于传统“单主多从”的固定组网方式，本作品支持根节点、中继节点、叶子节点三类角色协同工作，能够根据现场环境灵活扩展网络覆盖范围。  
这一设计更适合电力现场存在多柜体、多区域、多楼层、多支路的复杂部署条件，具备更好的工程适配能力。

### （5）统一网关汇聚上云，降低部署与运营成本
系统通过边缘网关对多节点数据进行集中处理和统一上云，避免每个采集点独立配备蜂窝通信模块和独立联网，大幅降低：

+  通信资费成本 
+  网络接入复杂度 
+  云端设备管理难度 
+  后期维护成本 

这种“**末端无线采集 + 边缘汇聚上云**”的模式更接近真实商用系统架构，具备更高的可复制性。

### （6）边缘计算提升现场实时性和自治能力
边缘网关可对采集数据进行本地规则分析，例如：

+  电流异常告警 
+  电压异常告警 
+  温湿度越限告警 
+  继电器联动控制 

在网络异常或云端不可达时，系统仍可保持本地基本监测和联动能力，增强现场自治水平。  
后续还可进一步扩展轻量化 AI 模型，实现趋势分析、异常预测和设备健康评分。

## 5. 作品的主要技术与产品优势
### （1）部署方式更符合真实工程需求
与需要大规模改动原有线路的方案相比，本作品支持在**不改变原有 485 仪表主体结构**的前提下进行无线化升级，更适合存量系统改造项目。

### （2）配置方式更友好，降低实施门槛
通过蓝牙、扫码、网页串口等多种方式实现参数配置，减少现场调试对复杂专用工具的依赖，提升工程实施效率。

### （3）充分发挥星闪特性，提升现场无线可靠性
本系统围绕星闪在速率、时延、抗干扰、跳频等方面的特性进行设计，使其在电力现场这种金属遮挡多、电磁干扰强、节点密集的环境下，更适合作为工业无线传输底座。

### （4）组网能力优于单层星型接入模式
系统采用多角色、多层级组网架构，相比单一“1 对 N”的固定接入方式，具备更强的扩展性、覆盖能力和场景适应性，更符合实际工业现场从“小规模验证”到“大范围部署”的演进需求。

### （5）网关统一汇聚，具备更好的商业可运营性
在商用落地中，单点独立联网方案虽然部署简单，但后续通信成本、设备管理复杂度和维护工作量较高。本系统采用统一网关汇聚模式，更有利于项目复制、集中运维和规模化推广。

### （6）边缘与云协同，更接近真实工业系统形态
系统既保留本地实时判断和联动能力，又支持云端进行趋势分析和远程管理，兼顾实时性、可靠性与智能化，符合当前工业数字化系统的主流建设思路。
