# 概述
通过触发器、执行条件和执行动作，构建一个完整的联动场景[温度超过28℃，打开空调](https://yunteng.yuque.com/avshoi/v2xdocs/cacgkugg1er2q4i8)。

# 触发器
:::color4
💡 提示

触发器不可为空，当场景联动中只有触发器时，消息只要满足触发条件即可触发。

触发类型详解：(触发器、执行条件、清除告警都一致)

简单：触发器和执行条件默认选中，无任何限制条件。

持续时长：可选秒、分、时、天。当设置为10秒持续时长时，在10秒内，推送的触发器值都满足，则场景联动生效。

重复次数：当数据达到触发器条件时，此时如果设置了重复次数，例如次数为3，则连续的3次数据都达到了触发器条件筛选值，则触发场景联动生效。

:::

## 属性触发
通过选择产品，设备和所创建的物模型属性，满足触发该属性所定义的范围即可触发，一个场景联动只能有一个触发器。

# 执行条件
:::color4
💡 提示

执行条件类似触发器，可为空，场景联动触发器满足后还必须满足全部的执行条件，场景联动才会执行生效。

:::

# 执行动作
触发器触发后，执行动作生效并回应，表示已经触发。

| **执行动作参数** | **参数说明** |
| --- | --- |
| 设备输出 | + 自定义命令输出和服务下发（TCP网关子没有服务功能） |
| 告警输出 | + 选择告警程度、选择告警配置[告警配置](https://yunteng.yuque.com/avshoi/v1xdocs/ixgfa5hst6ammzft)（非必选） |
| 清除告警 | + 选择告警输出后才可选清除告警，清除告警与触发器一 一对应 |
| 清除告警执行动作 | + 清除告警时，设备执行的输出动作，与设备输出一致 |


:::color4
💡 提示

触发器和执行条件都满足时，场景联动会做什么，例如：设备联动通知、告警通知、事件触发等。

:::

## 自定义输出
自定义执行动作，设备为MQTT设备下发为json数据格式，TCP设备下发为自定义命令。

类型：同步与异步。同步下发命令为双向，异步为单向。

## 告警输出
告警配置选择和告警等级选择。

:::color4
💡 提示

告警配置需提前配置好[告警配置](https://yunteng.yuque.com/avshoi/v1xdocs/ixgfa5hst6ammzft)（非必填）

:::

清除告警跟触发器一 一对应，触发器条件为温度大于30度产生告警，清除告警条件为温度小于30度即可。

清除告警执行动作：当告警满足清除条件时，下发到设备的命令记录。
