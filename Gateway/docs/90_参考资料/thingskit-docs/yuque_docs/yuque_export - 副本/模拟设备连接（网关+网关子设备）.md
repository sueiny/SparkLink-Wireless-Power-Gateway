## 2、发布、订阅主题和消息格式的构造
### 发布主题
主题内容：**v1/gateway/telemetry**

### 订阅主题
主题内容：**v1/devices/me/rpc/request/+**

### JSON内容
:::color4
💡注意：

1、MQTT协议

     支持：**Json**、**Protobuf**

     不支持：**HEX**、**CBOR**、**MsgPak**、**Plaintext**、**Base64**

2、以下为 **网关和网关子设备** 示例，需要其他设备示例，请参考 [模拟设备连接（直连）](https://yunteng.yuque.com/avshoi/v2xdocs/egntnkn55xvp1fwu)

:::

#### 数据格式A
数据格式A上报入库的时间以 **服务器时间 **为准。

```json
{
  "MQTT测试网关子设备1":[
    {
      "temperature":36.6
    }
  ]
}
```

```json
{
  "MQTT测试网关子设备1":[
    {
      "temperature":36.6
    }
  ],
  "MQTT测试网关子设备2":[
    {
      "temperature":35.6
    }
  ]
}
```

#### 数据格式B
数据格式B上报入库的时间以 **ts** 为准。这种格式推荐在进行** 数据补录 **时使用。

```json
{
  "MQTT测试网关子设备1": [
    {
      "ts": 1755517355236,
      "values": {
        "temperature": 42,
        "humidity": 80
      }
    },
    {
      "ts": 1755517387548,
      "values": {
        "temperature": 43,
        "humidity": 82
      }
    }
  ],
  "MQTT测试网关子设备2": [
    {
      "ts": 1755517387548,
      "values": {
        "temperature": 42,
        "humidity": 80
      }
    }
  ]
}
```
