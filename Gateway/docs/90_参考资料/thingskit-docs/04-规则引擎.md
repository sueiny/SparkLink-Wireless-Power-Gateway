# 规则引擎

## 概述

ThingsKit规则引擎基于图形化界面，通过拖拽方式创建和编排规则，实现数据过滤、转换、告警通知等功能。

## 规则组成

一个完整的规则由三部分组成：

```
触发器(Trigger) → 条件判断(Condition) → 执行动作(Action)
```

### 触发器类型

| 触发器 | 说明 | 示例 |
|--------|------|------|
| 设备触发 | 设备数据/事件触发 | 温度传感器上报数据 |
| 定时触发 | 定时/周期触发 | 每天8:00执行 |
| MQTT触发 | MQTT消息触发 | 监听指定Topic |
| HTTP触发 | HTTP请求触发 | API调用触发 |

### 条件判断

| 判断类型 | 说明 | 示例 |
|----------|------|------|
| 比较判断 | 数值比较 | 温度 > 30°C |
| 范围判断 | 范围匹配 | 20°C < 温度 < 30°C |
| 逻辑判断 | 与或非组合 | 温度>30 AND 湿度>80 |
| 字符串匹配 | 正则/包含 | 设备名包含"sensor" |
| 状态判断 | 设备状态 | 设备离线 |

### 执行动作

| 动作类型 | 说明 |
|----------|------|
| 发送通知 | 短信、邮件、微信、钉钉、Webhook |
| 设备控制 | 下发指令控制设备 |
| 数据转发 | 转发到其他Topic/平台 |
| 数据存储 | 存储到数据库 |
| 触发告警 | 创建告警记录 |
| 调用API | 调用外部接口 |

## 规则配置示例

### 示例1：温度超限告警
```yaml
name: 温度超限告警
trigger:
  type: device
  productId: temp_sensor
  dataPoint: temperature
condition:
  type: compare
  operator: ">"
  value: 35
action:
  - type: alarm
    level: warning
    message: "设备{{deviceName}}温度超标: {{temperature}}°C"
  - type: notify
    channel: [sms, email]
    template: "温度告警通知"
```

### 示例2：设备离线处理
```yaml
name: 设备离线通知
trigger:
  type: device
  event: offline
  duration: 5m
condition:
  type: group
  groupId: critical_devices
action:
  - type: notify
    channel: [wechat, dingtalk]
    template: "设备离线告警"
  - type: api
    url: https://api.example.com/device/offline
    method: POST
```

### 示例3：定时数据汇总
```yaml
name: 每小时数据汇总
trigger:
  type: cron
  expression: "0 */1 * * *"
condition:
  type: group
  groupId: all_sensors
action:
  - type: aggregate
    function: avg
    field: temperature
    output: hourly_avg_temp
  - type: store
    database: timeseries
```

## 规则编排

### 串行编排
```
触发 → 条件A → 条件B → 动作
```

### 并行编排
```
触发 → ┬ 条件A → 动作1
       └ 条件B → 动作2
```

### 条件分支编排
```
触发 → 条件判断 → 满足 → 动作A
                → 不满足 → 动作B
```

## 内置算子

### 数据处理算子
- **过滤器**：数据筛选
- **转换器**：数据格式转换
- **聚合器**：数据汇总计算
- **采样器**：数据采样

### 逻辑算子
- **AND**：逻辑与
- **OR**：逻辑或
- **NOT**：逻辑非
- **SWITCH**：多路分支

### 时间算子
- **延迟**：延时执行
- **定时**：定时触发
- **窗口**：时间窗口聚合

## 规则管理

### 规则状态
- **启用**：规则生效，实时执行
- **禁用**：规则暂停，不执行
- **错误**：规则配置错误，需修复
