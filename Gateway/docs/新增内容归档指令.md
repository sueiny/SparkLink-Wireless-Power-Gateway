# 新增内容归档指令

这份指引用于把 VS Code、WSL、语雀或临时目录中的新文档归档到 Gateway Obsidian vault。

Gateway vault 路径：

```text
D:\Dtu\Gateway
```

## 1. 先判断文档类型

新增文档不要直接堆到根目录。先判断它属于哪一类：

| 内容类型 | 归档位置 | 说明 |
| --- | --- | --- |
| 项目是什么、做什么、优势、讲解稿 | `docs/00_项目说明` | 面向答辩、简历、项目介绍 |
| 项目经历、阶段总结、问题解决 | `docs/10_项目复盘` | 面向复盘、面试回忆 |
| 线程、信号量、IPC、Socket、Modbus 等技术总结 | `docs/20_技术沉淀` | 面向技术沉淀 |
| 代码问题、架构问题、整改记录 | `docs/30_CodeReview` | 面向代码质量和重构 |
| 学习笔记、基础知识、技能路线 | `docs/40_学习技能` | 面向补基础和能力积累 |
| 外部资料、PDF、设备手册、语雀导出 | `docs/90_参考资料` | 默认不进入主图谱 |

## 2. 从 WSL/VS Code 归档 Markdown

如果文档在 WSL 中，例如：

```text
/home/sueiny/rk3506_linux6.1_v1.2.0/app/Gateway/docs/xxx.md
```

在 Windows PowerShell 中复制到 Gateway vault：

```powershell
Copy-Item `
  -LiteralPath "\\wsl.localhost\Ubuntu-22.04\home\sueiny\rk3506_linux6.1_v1.2.0\app\Gateway\docs\xxx.md" `
  -Destination "D:\Dtu\Gateway\docs\20_技术沉淀\xxx.md" `
  -Force
```

如果文档本来就在 Windows，例如：

```text
D:\Dtu\ob\xxx.md
```

复制：

```powershell
Copy-Item `
  -LiteralPath "D:\Dtu\ob\xxx.md" `
  -Destination "D:\Dtu\Gateway\docs\10_项目复盘\xxx.md" `
  -Force
```

## 3. 归档后必须更新索引

只复制文件还不够。要把它挂到对应索引，否则关系图谱里容易变成孤立节点。

| 归档目录 | 需要更新的索引 |
| --- | --- |
| `docs/00_项目说明` | `01-项目说明索引.md` |
| `docs/10_项目复盘` | `02-项目复盘索引.md` |
| `docs/20_技术沉淀` | `03-技术沉淀索引.md` |
| `docs/30_CodeReview` | `04-CodeReview索引.md` |
| `docs/40_学习技能` | `05-学习技能索引.md` |
| `docs/90_参考资料` | `06-参考资料索引.md` |

索引中添加一行：

```md
- [[docs/20_技术沉淀/线程设计|线程设计]]
```

如果文件名本身清晰，也可以不写别名：

```md
- [[docs/20_技术沉淀/线程设计]]
```

## 4. 常用归档命名规则

文件名建议短、明确、可搜索：

```text
线程设计.md
信号量设计.md
IPC通信.md
Socket通信.md
Modbus协议.md
数据源抽象.md
gatewayd代码Review.md
项目总复盘.md
问题与解决记录.md
```

避免：

```text
新建文档.md
未命名.md
最终版.md
最终版2.md
复制.md
```

## 5. 从语雀导入文档

语雀导出后优先使用 Markdown/ZIP。

建议先解压到临时目录：

```text
D:\Dtu\Gateway\_inbox\yuque
```

整理后再放入：

```text
D:\Dtu\Gateway\docs\90_参考资料\语雀
```

语雀全量导入不要直接挂入主索引，只挑选关键文档挂到 `06-参考资料索引.md`。

示例：

```text
docs/90_参考资料/语雀/设备MQTT接入.md
docs/90_参考资料/语雀/创建物模型.md
docs/90_参考资料/语雀/Docker单体部署V2.md
```

大量语雀资料应保持在参考区，避免污染主图谱。

## 6. 推荐复盘模板

新建技术沉淀文档时，建议使用这个结构：

```md
# 标题

## 项目中用在哪里

## 为什么需要它

## 具体设计

## 遇到的问题

## 怎么解决

## 后续可以怎么优化

## 面试/答辩怎么讲
```

新建项目复盘文档时，建议使用：

```md
# 标题

## 背景

## 目标

## 我负责的内容

## 关键问题

## 解决过程

## 最终结果

## 收获
```

## 7. 图谱维护规则

当前图谱主线是：

```text
00-Gateway总览
├─ 01-项目说明索引
├─ 02-项目复盘索引
├─ 03-技术沉淀索引
├─ 04-CodeReview索引
├─ 05-学习技能索引
└─ 06-参考资料索引
```

新增文档应该至少被一个索引链接。

不要把外部资料、语雀全量文档、PDF 手册全部挂到总览。总览只放主线。

## 8. 一次完整归档示例

目标：把 WSL 中的 `IPC通信总结.md` 归档为技术沉淀。

### 复制文件

```powershell
Copy-Item `
  -LiteralPath "\\wsl.localhost\Ubuntu-22.04\home\sueiny\rk3506_linux6.1_v1.2.0\app\Gateway\docs\IPC通信总结.md" `
  -Destination "D:\Dtu\Gateway\docs\20_技术沉淀\IPC通信.md" `
  -Force
```

### 更新索引

打开：

```text
D:\Dtu\Gateway\03-技术沉淀索引.md
```

加入：

```md
- [[docs/20_技术沉淀/IPC通信|IPC通信]]
```

### 检查

在 Obsidian 中打开 `00-Gateway总览`，确认可以通过：

```text
技术沉淀索引 -> IPC通信
```

找到文档。

## 9. 推荐工作流

1. 先写在 VS Code。
2. 文件完成后复制到对应分区。
3. 更新对应索引。
4. 如果是主线内容，再更新 `00-Gateway总览.md`。
5. 刷新 Obsidian 图谱，确认没有孤立节点。
