# Gateway (gatewayd) 后续架构规划

**版本**: v1.0  
**日期**: 2026-05-14  
**目标**: 规划 SLE 接入和 AI 接入两个阶段的架构设计

## 一、总体规划

### 1.1 两阶段路线

```
阶段 1: SLE 接入（当前 → 完成）
  ├─ 实现 sle-daemon 进程
  ├─ 实现 SLE 驱动 + 8 连接管理
  ├─ 实现 Unix Socket IPC
  ├─ 实现 Mesh + Modbus 解析
  └─ 替换 MockDataSource

阶段 2: AI 接入（SLE 完成后）
  ├─ 实现 ai-daemon 进程
  ├─ 实现 TFLite 推理引擎
  ├─ 实现特征提取 + 后处理
  ├─ 实现共享内存 IPC
  └─ 异常检测 + 智能告警
```

### 2.4 IPC 协议

```cpp
// SLE → gatewayd: 原始数据帧
struct SleRawFrame {
    uint8_t sle_connection_id;  // SLE 连接 ID (0-7)
    uint8_t mesh_node_id[8];    // Mesh 节点 ID
    uint8_t mesh_hop_count;     // 跳数
    uint8_t mesh_ttl;           // TTL
    uint16_t data_len;          // Modbus 数据长度
    uint8_t data[512];          // Modbus 原始数据
    int64_t timestamp_ms;       // 时间戳
};

// gatewayd → sle-daemon: 命令帧
struct SleCommandFrame {
    uint8_t target_mesh_node[8]; // 目标 Mesh 节点 ID
    uint16_t data_len;           // Modbus 请求长度
    uint8_t data[256];           // Modbus 请求数据
};
```

### 3.8 模型训练

```python
# Python 训练脚本 (在 PC 上运行)
import tensorflow as tf
import numpy as np

# 1. 准备训练数据
X_train = np.load("train_features.npy")  # (N, 12) 特征
y_train = np.load("train_labels.npy")    # (N,) 标签

# 2. 构建模型
model = tf.keras.Sequential([
    tf.keras.layers.Dense(64, activation='relu', input_shape=(12,)),
    tf.keras.layers.Dense(32, activation='relu'),
    tf.keras.layers.Dense(16, activation='relu'),
    tf.keras.layers.Dense(1, activation='sigmoid')
])

model.compile(optimizer='adam', loss='binary_crossentropy', metrics=['accuracy'])

# 3. 训练
model.fit(X_train, y_train, epochs=50, batch_size=32, validation_split=0.2)

# 4. 转换为 TFLite
converter = tf.lite.TFLiteConverter.from_keras_model(model)
converter.optimizations = [tf.lite.Optimize.DEFAULT]
tflite_model = converter.convert()

# 5. 保存
with open("anomaly_detector.tflite", "wb") as f:
    f.write(tflite_model)
```

## 六、风险评估

### 6.1 SLE 阶段风险

| 风险 | 概率 | 影响 | 缓解措施 |
|------|------|------|---------|
| SLE 驱动崩溃 | 中 | 丢失 SLE 数据 | 进程隔离 + 自动重启 |
| Mesh 解析错误 | 低 | 数据错误 | 完善日志 + 校验 |
| IPC 数据丢失 | 低 | 丢失单帧数据 | 应用层重传 |
| 内存不足 | 低 | 系统变慢 | 监控内存使用 |

### 6.2 AI 阶段风险

| 风险 | 概率 | 影响 | 缓解措施 |
|------|------|------|---------|
| 模型推理超时 | 中 | AI 结果延迟 | 降低推理频率 |
| 内存不足 | 低 | 系统变慢 | 减少历史缓存 |
| 模型精度不足 | 中 | 误报/漏报 | 持续优化模型 |
| 共享内存竞争 | 低 | 数据不一致 | 使用原子操作 |
