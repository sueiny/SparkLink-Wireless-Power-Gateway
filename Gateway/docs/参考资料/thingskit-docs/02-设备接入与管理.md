# 设备接入与管理

## 支持的接入协议

| 协议 | 说明 | 适用场景 |
|------|------|----------|
| MQTT | 轻量级消息协议 | 传感器、低带宽设备 |
| HTTP | RESTful API | 云端对接、Web应用 |
| TCP | 原始TCP连接 | 自定义协议设备 |
| UDP | 无连接协议 | 实时性要求高的场景 |
| CoAP | 受限应用协议 | 低功耗物联网设备 |
| GB28181 | 国标视频协议 | 摄像头、视频设备 |

## 产品物模型

物模型是设备的数字化描述，定义设备的功能特性：

### 物模型组成
- **属性（Property）**: 设备的状态数据，如温度、湿度、开关状态
- **事件（Event）**: 设备主动上报的信息，如告警、故障
- **服务（Service）**: 设备可被调用的功能，如重启、配置更新

### 物模型定义示例
```json
{
  "productId": "temp_sensor_001",
  "properties": [
    {
      "id": "temperature",
      "name": "温度",
      "type": "double",
      "unit": "°C",
      "accessMode": "r"
    },
    {
      "id": "humidity",
      "name": "湿度",
      "type": "double",
      "unit": "%",
      "accessMode": "r"
    }
  ],
  "events": [
    {
      "id": "overheat",
      "name": "过热告警",
      "type": "alert"
    }
  ],
  "services": [
    {
      "id": "setReportInterval",
      "name": "设置上报周期",
      "params": [
        {
          "id": "interval",
          "type": "int",
          "unit": "ms"
        }
      ]
    }
  ]
}
```

## 设备生命周期

```
注册 → 认证 → 激活 → 在线 ↔ 离线 → 注销
```

### 各阶段说明

| 阶段 | 说明 |
|------|------|
| 注册 | 在平台创建设备，获取设备凭证 |
| 认证 | 设备携带凭证连接平台 |
| 激活 | 首次认证成功，设备激活 |
| 在线 | 设备正常连接，数据可上报 |
| 离线 | 设备断开连接 |
| 注销 | 从平台移除设备 |

## 设备管理功能

### 设备分组
- 按区域、类型、用途分组管理
- 支持多级分组结构
- 批量操作支持

### 标签管理
- 自定义标签键值对
- 支持按标签筛选设备
- 便于批量管理和规则匹配

### 设备影子
```json
{
  "deviceId": "sensor_001",
  "reported": {
    "temperature": 25.5,
    "humidity": 60.2
  },
  "desired": {
    "reportInterval": 5000
  }
}
```
- 缓存设备最新状态
- 支持离线状态查询
- 实现期望状态与实际状态同步

## 远程控制

### 指令下发
```json
{
  "method": "set",
  "params": {
    "switch": true,
    "brightness": 80
  }
}
```

### 支持的控制方式
- 同步调用：等待设备响应
- 异步调用：下发后立即返回
- 批量控制：同时控制多台设备

## 固件升级（OTA）

### 升级流程
1. 上传固件包到平台
2. 创建升级任务
3. 选择目标设备/分组
4. 设备拉取固件
5. 设备执行升级
6. 上报升级结果

### 支持的升级模式
- 全量升级
- 灰度升级（按比例/分组）
- 差分升级
