---
title: 告警记录
---

# 概述
当设备达到预先指定的阈值时，平台会自动产生告警，可以通过告警记录及时查看详细的告警信息以及对告警进行处理和反馈。

:::color4
💡注意：

需要先配置场景联动，配置触发器，执行动作。

:::

# 告警详情
场景联动中设备产生的告警记录详情。

:::color4
💡注意：

告警记录状态：激活未确认、激活已确认、清除未确认、清除已确认。

激活未确认指告警没有处理，清除，存在告警。激活已确认指告警没有清除，已经处理，还是存在告警。

清除未确认指告警已经清除，场景联动里面配置清除告警。清除已确认指已经不存在告警风险。

:::

💡注意：

场景联动中配置清除告警，在记录为激活未确认时，满足清除告警条件，则记录状态变为清除未确认。

同一状态的才可以进行批量操作，比如都是激活未确认都可以进行清除和处理，激活已确认和清除未确认这样就不能进行批量操作。

:::
