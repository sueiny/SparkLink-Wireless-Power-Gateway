## 2、发布、订阅主题和消息格式的构造
### 发布主题
主题内容：**v1/devices/me/telemetry**

### 订阅主题
主题内容：**v1/devices/me/rpc/request/+**

### JSON内容
:::color4
💡注意：

1、MQTT协议

     支持：**Json**、**Protobuf**

     不支持：**HEX**、**CBOR**、**MsgPak**、**Plaintext**、**Base64**

2、以下为 **直连设备** 示例，需要其他设备示例，请参考 [**网关和网关子设备上报示例**](https://yunteng.yuque.com/avshoi/v2xdocs/tb3whyrsf7gkyrcw)

:::

#### 数据格式A
数据格式A上报入库的时间以 **服务器时间 **为准。

```json
{
    "CurrentTemperature":26.8,
    "CurrentHumidity":65.9,
    "APValue":990
}
```

```json
[{
    "CurrentTemperature":26.8,
    "CurrentHumidity":65.9,
    "APValue":990
},
{
    "CurrentTemperature":26.9,
    "CurrentHumidity":66,
    "APValue":995
}]
```

#### 数据格式B
数据格式B上报入库的时间以 **ts** 为准。这种格式推荐在进行** 数据补录 **时使用。

```json
{
	"ts": 1755174230208,
	"values": {
		"CurrentTemperature": 26.8,
		"CurrentHumidity": 65.9,
		"APValue": 990
	}
}
```
