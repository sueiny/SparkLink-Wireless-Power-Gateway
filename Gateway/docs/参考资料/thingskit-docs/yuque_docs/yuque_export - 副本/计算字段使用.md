---
title: 计算字段使用
---

<font style="color:rgb(33, 37, 41);">允许租户管理员对遥测数据和属性执行实时计算，从而实现无缝数据转换，无需规则链和额外的逻辑块。通过定义自定义表达式或脚本，用户可以标准化数据，并动态创建新的计算指标。此功能对于优化数据处理和改进分析尤其有用。</font>

:::color4
💡注意：

<font style="color:rgb(33, 37, 41);">只有租户才有权配置和管理计算字段</font>

:::

# 前置条件
产品物模型也已经正确创建，同时设备已经正常创建并上线，如需了解请参考[创建物模型](https://yunteng.yuque.com/avshoi/v2xdocs/toqngtefr0yeq2pi) 和 [数据上报（模拟）](https://yunteng.yuque.com/avshoi/v2xdocs/bnxw31sunnh1ewt2)。

## **<font style="color:rgb(33, 37, 41);">主要优点</font>**
+ **<font style="color:rgb(33, 37, 41);">规则链中无需额外逻辑</font>**<font style="color:rgb(33, 37, 41);">：简化遥测计算，无需复杂的规则链配置。</font>
+ **<font style="color:rgb(33, 37, 41);">实时计算：</font>**<font style="color:rgb(33, 37, 41);">当传入的遥测和属性由保存时间序列、</font>[<font style="color:rgb(33, 37, 41);">保存属性</font>](https://thingsboard.io/docs/user-guide/rule-engine-2-0/action-nodes/#save-attributes-node)<font style="color:rgb(33, 37, 41);">或</font>[<font style="color:rgb(33, 37, 41);">计算字段</font>](https://thingsboard.io/docs/user-guide/rule-engine-2-0/action-nodes/#calculated-fields-node)<font style="color:rgb(33, 37, 41);">规则节点处理时触发计算，确保获得最新的见解。</font>
+ **<font style="color:rgb(33, 37, 41);">优化性能</font>**<font style="color:rgb(33, 37, 41);">：通过在接收数据时执行计算来减少数据库查询，从而提高系统效率。</font>
+ **<font style="color:rgb(33, 37, 41);">跨实体数据合并</font>**<font style="color:rgb(33, 37, 41);">：通过组合来自多个来源（设备、资产等）的数据来计算新值。</font>
+ **<font style="color:rgb(33, 37, 41);">灵活的输出</font>**<font style="color:rgb(33, 37, 41);">：根据用例，将结果存储为</font>[<font style="color:rgb(33, 37, 41);">属性</font>](https://thingsboard.io/docs/user-guide/attributes/)<font style="color:rgb(33, 37, 41);">或时间序列数据。</font>

## <font style="color:rgb(33, 37, 41);"></font>**<font style="color:rgb(33, 37, 41);">用例示例</font>**
+ **<font style="color:rgb(33, 37, 41);">结合来自多个来源的遥测数据</font>**<font style="color:rgb(33, 37, 41);">：根据设备 A 的温度和设备 B 的湿度等计算露点。</font>
+ **<font style="color:rgb(33, 37, 41);">标准化测量单位</font>**<font style="color:rgb(33, 37, 41);">：将温度读数从摄氏度转换为华氏度，或标准化不同传感器模型之间的压力和电压水平。</font>
+ **<font style="color:rgb(33, 37, 41);">能源消耗跟踪</font>**<font style="color:rgb(33, 37, 41);">：计算每小时的用电量，以帮助优化能源消耗和成本效率。</font>
+ **<font style="color:rgb(33, 37, 41);">数据平滑</font>**<font style="color:rgb(33, 37, 41);">：计算湿度或温度等环境遥测的滚动平均值，以减少传感器读数的波动。</font>
+ **<font style="color:rgb(33, 37, 41);">预测性维护</font>**<font style="color:rgb(33, 37, 41);">：生成机器的效率指标，例如空气密度计算，以预测故障发生前的维护需求。</font>
+ **<font style="color:rgb(33, 37, 41);">自定义业务逻辑</font>**<font style="color:rgb(33, 37, 41);">：根据特定业务需求实施高级计算，例如根据运动传感器数据确定占用水平或根据多个遥测输入动态调整设备设置。</font>

### 创建计算字段
计算字段参数说明如下所示：

+ **简单**：进行一些简单的计算，例如：+ - * / % ^等，可参考[简单计算](https://yunteng.yuque.com/avshoi/v2xdocs/math-methods_fn)帮助文档
+ **脚本**：通过脚本进行一些复杂的计算，可以参考[脚本计算](https://yunteng.yuque.com/avshoi/v2xdocs/expression_fn)帮助文档
+ **参数**：参与计算设备的字段，并给这些选择的字段重新命名一个新的名称，用于**表达式**的计算或者**脚本**的计算
+ **输出类型**：遥测，即输出到**遥测数据**。属性，可输出到**服务端属性**和**共享属性**中。
+ **输出字段**：物模型的标识符或者属性的key
