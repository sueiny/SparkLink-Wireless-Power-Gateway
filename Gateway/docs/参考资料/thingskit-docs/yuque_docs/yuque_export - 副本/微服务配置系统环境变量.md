# 第1/7步(可选)：配置名称
:::info
💡 提示

用户名【USER_NAME】和对象名【PGSQL_OBJ_NAME】不能包含特殊字符。

:::

## 编辑配置文件
```shell
sed -i 's/USER_NAME=thingskit/USER_NAME=你的用户名/g' thingskit2.0.sh
```

```shell
sed -i 's/PGSQL_OBJ_NAME=thingskit/USER_NAME=你的对象名/g' thingskit2.0.sh
```

## 查看结果
```shell
cat thingskit2.0.sh|grep thingskit
```

:::danger
⚠️ 警告

如果还可以找到文本【thingskit】说明配置文件编辑失败。

:::

# 第2/7步：配置密码
:::info
💡 提示

建议使用满足密码规范的密码。

:::

## 编辑配置文件
```shell
echo $(openssl rand -base64 32 | tr -dc 'a-zA-Z0-9!@#$%^&*()_+-=' | head -c 16)
```

```shell
sed -i "s|默认密码|新密码|" thingskit2.0.sh
```

## 查看结果
```shell
cat thingskit2.0.sh|grep 默认密码
```

:::danger
⚠️ 警告

如果还可以找到文本【默认密码】说明配置文件编辑失败。

:::

# 第3/7步：配置平台访问信息


## 编辑配置文件
```shell
cd /_makeFile
```

```shell
sed -i '/PLATFORM_SERVICE_DOMAIN=/c\export PLATFORM_SERVICE_DOMAIN=你的平台访问域名或IP/g' thingskit2.0.sh
sed -i '/PLATFORM_SERVICE_PORT=/c\export PLATFORM_SERVICE_PORT=你的平台访问端口/g' thingskit2.0.sh

```



## 查看结果
```shell
grep PLATFORM_SERVICE_ thingskit2.0.sh
```

:::danger
⚠️ 警告

如果【PLATFORM_SERVICE_】相关变量的值没有跟新，说明配置文件编辑失败。

:::

# 第6/7步(可选)：第三方软件端口修改
:::danger
⚠️ 警告

对于存在漏洞的第三方软件存在安全风险，强烈建议使用非标准的端口。

:::

## 编辑配置文件


```shell
sed -i "s/ZOOKEEPER_SERVICE_PORT=2181/ZOOKEEPER_SERVICE_PORT=你的端口/" thingskit2.0.sh
```

```shell
sed -i "s/REDIS_SERVICE_PORT=6379/REDIS_SERVICE_PORT=你的端口/" thingskit2.0.sh
```

```shell
sed -i "s/KAFKA_SERVICE_PORT=9092/KAFKA_SERVICE_PORT=你的端口/" thingskit2.0.sh
```

```shell
sed -i "s/PG_SERVICE_PORT=5432/PG_SERVICE_PORT=你的端口/" thingskit2.0.sh
```

## 查看结果
```shell
cat thingskit2.0.sh|grep _PORT
```

:::danger
⚠️ 警告

如果对应端口内容没有更新，说明配置文件编辑失败。

:::
