# 脚本初始化服务器信息
## 执行脚本
:::info
💡提示

如果source命令异常，可以重启系统使thingskit2.0.sh生效。

:::

:::info
💡提示

如果在执行init_all.sh脚本时填错内容，可以在thingskit2.0.sh内进行修改。

:::

:::info
💡提示

如果在执行init_all.sh脚本时填错内容，可以在thingskit2.0.sh内进行修改。修改后需要重新执行source /etc/profile。

:::

```shell
cd /_makeFile
chmod +x init_one.sh
./init_one.sh

cd /_makeFile
mv thingskit2.0.sh /etc/profile.d
source /etc/profile						#使脚本获取到的服务器信息生效后 才能执行后续脚本

```

## 查看结果
```shell
echo $CLUSTER_NODE_ID
```

:::danger
⚠️ 警告

如果负载均衡内容未刷新，说明脚本执行失败。可以考虑手动操作。

:::

# 第1/3步(可选)：修改软件服务的监听IP
## 编辑配置文件
:::info
💡 提示

默认监听网卡IP。

:::

```shell
cd /etc/profile.d

sed -i '/SERVER_HOST_DOMAIN=/c\export SERVER_HOST_DOMAIN=当前节点的公网IP/g' thingskit2.0.sh

```

## 查看结果
```shell
grep SERVER_HOST_DOMAIN thingskit2.0.sh
```

:::danger
⚠️ 警告

如果【SERVER_HOST_DOMAIN】的值为更新，说明配置文件编辑失败。

:::

# 第2/3步：修改节点ID
:::info
💡 提示

环境变量【CLUSTER_NODE_ID】默认配置为1，服务器节点1不需要执行改步操作。

:::

## 编辑配置文件
```shell
cd /etc/profile.d

sed -i "/CLUSTER_NODE_ID/c\CLUSTER_NODE_ID=服务节点的数字编号" thingskit2.0.sh
```

## 查看结果
```shell
grep "CLUSTER_NODE_ID" thingskit2.0.sh
```

:::danger
⚠️ 警告

如果变量【CLUSTER_NODE_ID】的值未刷新说明编辑失败。

:::

# 第3/3步：修改服务器节点IP为广播地址
## 编辑配置文件
```shell
cd /_makeFile
```

:::info
💡 提示

【集群节点01】需要替换为【对应节点的网卡(内网)IP】后执行命令。

:::

:::info
💡 提示

命令中的【CLUSTER_NODE_ONE】也需要根据节点ID进行修改。

可选值：CLUSTER_NODE_ONE、CLUSTER_NODE_TWO、CLUSTER_NODE_THREE、CLUSTER_NODE_FOUR、CLUSTER_NODE_FIVE。

:::

```shell
sed -i 's#\${CLUSTER_NODE_ONE}#0.0.0.0#g' miscroservice_zookeeper.env
```

## 查看结果
```shell
grep "0.0.0.0" miscroservice_zookeeper.env
```

:::danger
⚠️ 警告

如果文本中找不到【0.0.0.0】说明配置文件编辑失败。

:::
