# 边缘网关开发指南

## 概述

边缘网关是云边协同架构的核心组件，负责连接终端设备与云端平台，实现数据采集、预处理、本地决策和云边同步。

## 网关核心功能

### 1. 设备接入管理
- 多协议设备接入（Modbus、OPC UA、BACnet等）
- 协议转换：将私有协议转换为平台标准协议
- 设备发现与自动注册

### 2. 数据采集与处理
```
设备 → 数据采集 → 数据清洗 → 数据聚合 → 上报云端
```

#### 数据采集
- 轮询采集
- 变化上报
- 定时采集

#### 数据预处理
- 数据过滤：去除无效/异常数据
- 数据聚合：多点数据合并
- 数据转换：单位换算、格式转换
- 边缘计算：本地统计、告警判断

### 3. 本地决策
- 边缘规则执行
- 本地告警触发
- 设备联动控制
- 断网情况下的自主运行

### 4. 云边同步

#### 数据上行
- 设备数据上报到云端
- 支持批量上报
- 数据压缩传输

#### 指令下行
- 云端指令下发到网关
- 指令缓存与重发
- 离线指令队列

#### 配置同步
- 物模型配置下发
- 规则配置同步
- 设备列表同步

### 5. 离线自治
```
正常模式 ↔ 离线模式
     ↓
本地缓存 → 断点续传
```

- 离线数据本地存储
- 网络恢复后自动补传
- 本地规则持续运行

## 网关接入协议

### MQTT接入（推荐）

#### 连接配置
```json
{
  "broker": "mqtt://thingskit.com:1883",
  "clientId": "gateway_001",
  "username": "gateway_001",
  "password": "****",
  "keepalive": 60,
  "cleanSession": true
}
```

#### Topic规范
```
# 数据上行
devices/{gatewayId}/telemetry        # 遥测数据
devices/{gatewayId}/attributes       # 属性数据
devices/{gatewayId}/events           # 事件上报

# 指令下行
devices/{gatewayId}/commands/request  # 指令下发
devices/{gatewayId}/commands/response # 指令响应

# 设备管理
devices/{gatewayId}/status           # 设备状态
```

#### 数据上报格式
```json
{
  "deviceId": "sensor_001",
  "ts": 1699000000000,
  "values": {
    "temperature": 25.5,
    "humidity": 60.2
  }
}
```

### HTTP接入

#### 数据上报API
```
POST /api/v1/gateway/telemetry
Content-Type: application/json
Authorization: Bearer {token}

{
  "gatewayId": "gateway_001",
  "devices": [
    {
      "deviceId": "sensor_001",
      "data": {
        "temperature": 25.5,
        "humidity": 60.2
      }
    }
  ]
}
```

### TCP/UDP接入
- 自定义协议解析
- 粘包/拆包处理
- 心跳保活机制

## Modbus协议接入

### 寄存器映射
```json
{
  "slaveId": 1,
  "registers": [
    {
      "name": "temperature",
      "address": 0,
      "type": "holding",
      "dataType": "float",
      "scale": 0.1,
      "unit": "°C"
    },
    {
      "name": "humidity",
      "address": 2,
      "type": "holding",
      "dataType": "float",
      "scale": 0.1,
      "unit": "%"
    }
  ]
}
```

### 采集配置
```json
{
  "protocol": "modbus_rtu",
  "serialPort": "/dev/ttyS0",
  "baudRate": 9600,
  "dataBits": 8,
  "stopBits": 1,
  "parity": "none",
  "pollInterval": 5000
}
```

## OPC UA接入

### 连接配置
```json
{
  "endpoint": "opc.tcp://192.168.1.100:4840",
  "securityMode": "None",
  "authMode": "Anonymous",
  "nodes": [
    {
      "nodeId": "ns=2;s=Temperature",
      "name": "temperature",
      "samplingInterval": 1000
    }
  ]
}
```

## 网关部署与管理

### 部署方式
1. **Docker容器部署**
```bash
docker run -d \
  --name thingskit-edge \
  -v /etc/thingskit:/config \
  -p 1883:1883 \
  thingskit/edge:latest
```

2. **系统服务部署**
```bash
sudo systemctl start thingskit-edge
sudo systemctl enable thingskit-edge
```

## 开发建议

### 协议适配
1. 实现标准MQTT客户端
2. 遵循平台Topic规范
3. 使用平台数据格式
4. 处理连接异常与重连

### 数据可靠性
1. 本地数据缓存
2. 上报失败重试
3. 断点续传机制
4. 数据去重处理

### 安全性
1. TLS/SSL加密传输
2. 设备认证鉴权
3. 固件签名验证
4. 安全启动
