### 阶段 1：线程架构 — Worker 模式

**对应工程任务**：P0 线程架构清晰化

**学到的核心概念**：

| 概念 | 说明 | 本项目应用 |
|------|------|-----------|
| Worker 模式 | 每个工作单元封装为独立类 | CollectWorker, NetworkWorker, PublishWorker, CommandWorker |
| 线程生命周期 | start → run → stop → join | 统一 Worker 接口 |
| 优雅退出 | atomic_bool + condition_variable + stop() | 确保所有线程干净退出 |
| 线程命名 | pthread_setname_np | 调试时 ps -T 看线程名 |

**关键代码模式**：

```cpp
// Worker 接口
class Worker {
public:
    virtual ~Worker() = default;
    virtual void start() = 0;   // 创建线程
    virtual void stop() = 0;    // 通知退出
    virtual void join() = 0;    // 等待结束
    virtual const char* name() const = 0;
};

// 典型实现
class CollectWorker : public Worker {
public:
    void start() override {
        thread_ = std::thread([this] { run(); });
    }
    void stop() override { quit_.store(true); cv_.notify_all(); }
    void join() override { if (thread_.joinable()) thread_.join(); }
    const char* name() const override { return "collect"; }
    
private:
    void run() {
        while (!quit_.load()) {
            // 采集逻辑
            cv_.wait_for(lock, interval, [this]{ return quit_.load(); });
        }
    }
    
    std::thread thread_;
    std::atomic_bool quit_{false};
    std::condition_variable cv_;
    std::mutex mutex_;
};
```

**学习收获**：
- 理解线程的完整生命周期
- 掌握 Worker 模式，后续任何多线程项目都能复用
- 学会用 condition_variable 实现定时器和优雅退出

### 阶段 2：进程管理 — fork/exec/signals

**对应工程任务**：P1-Step0 runProcess() 基础设施

**学到的核心概念**：

| 概念 | 说明 | 本项目应用 |
|------|------|-----------|
| fork() | 创建子进程 | 执行 udhcpc/wpa_cli/pppd |
| execvp() | 替换进程映像 | 在子进程中执行外部命令 |
| waitpid() | 等待子进程结束 | 获取退出状态 |
| alarm() | 设置定时信号 | 实现超时控制 |
| pipe() | 进程间通信 | 捕获子进程 stdout |
| SIGCHLD | 子进程退出信号 | 异步回收子进程 |

**关键代码模式**：

```cpp
ProcessResult runProcess(const std::vector<std::string> &args, int timeout_ms) {
    int stdout_pipe[2];
    pipe(stdout_pipe);
    
    pid_t pid = fork();
    if (pid == 0) {
        // 子进程
        close(stdout_pipe[0]);
        dup2(stdout_pipe[1], STDOUT_FILENO);
        
        std::vector<char*> argv;
        for (const auto &arg : args) argv.push_back(const_cast<char*>(arg.c_str()));
        argv.push_back(nullptr);
        execvp(argv[0], argv.data());
        _exit(127);  // exec 失败
    }
    
    // 父进程
    close(stdout_pipe[1]);
    
    // 带超时等待
    auto deadline = steady_clock::now() + milliseconds(timeout_ms);
    int status;
    while (steady_clock::now() < deadline) {
        pid_t result = waitpid(pid, &status, WNOHANG);
        if (result > 0) {
            // 读取 stdout
            // 返回结果
        }
        this_thread::sleep_for(milliseconds(50));
    }
    
    // 超时，kill 子进程
    kill(pid, SIGKILL);
    waitpid(pid, &status, 0);
    return {-1, "", true};
}
```

**学习收获**：
- 理解 fork 的"一次调用，两次返回"
- 掌握 exec 家族函数的用法
- 学会用 pipe 捕获子进程输出
- 理解信号机制（SIGCHLD, SIGALRM, SIGKILL）

**延伸阅读**：
- `man 2 fork`
- `man 2 execve`
- `man 2 waitpid`
- `man 2 pipe`

bool setInterfaceUp(const std::string &ifname) {
    int fd = socket(AF_INET, SOCK_DGRAM, 0);
    
    struct ifreq ifr {};
    strncpy(ifr.ifr_name, ifname.c_str(), IFNAMSIZ - 1);
    
    // 读取当前标志
    ioctl(fd, SIOCGIFFLAGS, &ifr);

    ifr.ifr_flags |= IFF_UP | IFF_RUNNING;

}

### 阶段 5：串口编程 — Termios + AT 指令

**对应工程任务**：P1-Step3 CellularProvider

**学到的核心概念**：

| 概念 | 说明 | 本项目应用 |
|------|------|-----------|
| termios | 终端 I/O 控制 | 配置串口参数 |
| 波特率 | 数据传输速率 | 115200 bps |
| AT 指令 | 调制解调器控制协议 | 拨号/查询状态 |
| 请求-响应模式 | 发命令等回复 | AT 协议交互 |

**串口配置**：

```cpp
int openSerial(const std::string &device, int baudrate) {
    int fd = open(device.c_str(), O_RDWR | O_NOCTTY | O_NONBLOCK);
    
    struct termios tty {};
    tcgetattr(fd, &tty);
    
    // 设置波特率
    cfsetospeed(&tty, B115200);
    cfsetispeed(&tty, B115200);
    
    // 8N1 (8 数据位, 无校验, 1 停止位)
    tty.c_cflag &= ~PARENB;   // 无校验
    tty.c_cflag &= ~CSTOPB;   // 1 停止位
    tty.c_cflag &= ~CSIZE;
    tty.c_cflag |= CS8;        // 8 数据位
    tty.c_cflag |= CLOCAL | CREAD;  // 本地模式, 允许读取
    
    tcsetattr(fd, TCSANOW, &tty);
    return fd;
}
```

**AT 指令交互**：

```cpp
bool sendAtCommand(int fd, const std::string &cmd, 
                   const std::string &expected, int timeout_ms) {
    // 发送命令
    std::string full_cmd = cmd + "\r";
    write(fd, full_cmd.c_str(), full_cmd.size());
    
    // 读取响应，带超时
    char buf[256];
    std::string response;
    auto deadline = steady_clock::now() + milliseconds(timeout_ms);
    
    while (steady_clock::now() < deadline) {
        struct pollfd pfd {fd, POLLIN, 0};
        if (poll(&pfd, 1, 200) > 0) {
            int n = read(fd, buf, sizeof(buf) - 1);
            if (n > 0) {
                buf[n] = '\0';
                response += buf;
                if (response.find(expected) != std::string::npos)
                    return true;   // 收到期望响应
                if (response.find("ERROR") != std::string::npos)
                    return false;  // 收到错误
            }
        }
    }
    return false;  // 超时
}
```

**典型 AT 指令序列**：

```
AT                  → OK           (握手)
AT+CPIN?            → +CPIN: READY (SIM 卡就绪)
AT+CSQ              → +CSQ: 20,0   (信号强度)
AT+CGDCONT=1,"IP","cmnet" → OK     (设置 APN)
ATD*99#             → CONNECT      (拨号)
```

    unlink(oldest.c_str());

class FileWatcher {
public:
    bool watch(const std::string &path) {
        inotify_fd_ = inotify_init();
        watch_fd_ = inotify_add_watch(inotify_fd_, path.c_str(), IN_MODIFY | IN_DELETE_SELF);

};
