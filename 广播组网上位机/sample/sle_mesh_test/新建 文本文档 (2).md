# Mesh Serial Protocol

## 1. ACK (0x05) — 到达确认

用于节点收到 DATA 后返回确认消息。

### 数据结构

| 偏移 | 字段名 | 长度 | 说明 |
|---|---|---|---|
| 0 | magic | 2 | 固定值 `0x4D 0x53` |
| 2 | version | 1 | 协议版本，当前为 `0x01` |
| 3 | type | 1 | 消息类型 `0x05` |
| 4 | src_addr | 1 | ACK 发送者（目的节点） |
| 5 | dst_addr | 1 | 固定为 `0x00`（dongle） |
| 6 | seq | 2 | 与原始 DATA 的 `seq` 匹配 |

---

## ACK 示例

假设：

- 节点 `0x12` 收到了 seq=`0x0034` 的 DATA
- 返回 ACK 给 dongle (`0x00`)

### 十六进制数据

```text
4D 53 01 05 12 00 00 34
```

### 字段解析

| 字段 | 值 |
|---|---|
| magic | `4D 53` |
| version | `01` |
| type | `05` |
| src_addr | `12` |
| dst_addr | `00` |
| seq | `00 34` |

---

# 2. PC 串口下发格式

PC 向 dongle 发送数据时使用以下格式：

```text
SEND <dst> <path_len> <addr0> <addr1> ... <hex_payload>\r\n
```

## 字段说明

| 字段 | 说明 |
|---|---|
| dst | 最终目标节点地址 |
| path_len | 路径长度 |
| addr0 ~ addrN | 中继路径节点地址 |
| hex_payload | 十六进制 Payload 数据 |
| `\r\n` | 行结束符 |

---

## SEND 示例

目标节点：

```text
0x12
```

路由路径：

```text
0x05 -> 0x08 -> 0x12
```

Payload：

```text
01 02 03 04
```

### PC 输入

```text
SEND 12 3 05 08 12 01020304
```

# 3. RSSI_REPORT (0x01) — 节点 → dongle → PC

用于节点上报邻居 RSSI 信息。

## 数据结构

| 偏移 | 字段 | 长度 | 说明 |
|---|---|---|---|
| 0 | magic | 2 | 固定值 `0x4D 0x53` |
| 2 | version | 1 | 协议版本 `0x01` |
| 3 | type | 1 | 消息类型 `0x01` |
| 4 | src_addr | 1 | 上报节点地址 |
| 5 | count | 1 | 邻居数量 |
| 6 | [addr, rssi] | 2 × N | 邻居地址 + RSSI 对 |

---

## RSSI_REPORT 示例

节点 `0x12` 上报两个邻居：

| 邻居地址 | RSSI |
|---|---|
| `0x05` | `-45 dBm` |
| `0x08` | `-60 dBm` |

假设 RSSI 使用 int8 编码：

- `-45 = 0xD3`
- `-60 = 0xC4`

### 十六进制数据

```text
4D 53 01 01 12 02 05 D3 08 C4
```

---

## 字段解析

| 字段 | 值 |
|---|---|
| magic | `4D 53` |
| version | `01` |
| type | `01` |
| src_addr | `12` |
| count | `02` |
| neighbor #1 | `05 D3` |
| neighbor #2 | `08 C4` |

---

# RSSI 编码说明

RSSI 使用：

```text
int8_t
```

即：

- `0xD3` = `-45`
- `0xC4` = `-60`

单位：

```text
dBm
```
