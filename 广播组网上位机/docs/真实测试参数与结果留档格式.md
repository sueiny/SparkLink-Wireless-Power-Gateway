# 真实测试参数与结果留档格式

本文定义真实硬件测试时，延时、丢包率、命令间隔、RSSI 拓扑和当前 `sle_mesh_test` 广播参数如何单独留档。该文档对应一份独立输出文件，建议文件名：

```text
hardware_test_record.json
```

这份文件用于保证每次测试可追溯：同一份延时和丢包率结果必须能回看当时的串口参数、节点列表、命令间隔、RSSI 采集设置、广播/扫描参数、Dijkstra 权重规则和默认 D3QN 参数。

## 1. 顶层结构

推荐顶层结构如下：

```json
{
  "schema": "hardware_test_record.v1",
  "run_id": "formal_v1",
  "algorithm": "dijkstra",
  "source_log_dir": "app/广播组网上位机/app/logs/dijkstra_hw/formal_v1",
  "test_config": {},
  "firmware_params": {},
  "routing_params": {},
  "topology_summary": {},
  "result_summary": {},
  "field_sources": {}
}
```

其中：

- `test_config` 记录 PC 上位机测试参数。
- `firmware_params` 记录 `sample/sle_mesh_test` 当前广播、扫描、缓存、去重等固件参数。
- `routing_params` 记录 Dijkstra 权重规则。
- `topology_summary` 记录 RSSI 拓扑摘要。
- `result_summary` 记录真实 ACK 延时和丢包率。
- `field_sources` 记录字段来源。

## 7. 字段来源

所有结果必须带来源说明：

| 来源 | 含义 |
|---|---|
| `real_serial` | 串口真实 TX/RX 原始数据 |
| `real_rssi` | `RSSI_REQ` 后由 `RSSI_REPORT` 得到 |
| `real_ack` | `SEND` 后由真实 `ACK` 或 timeout 得到 |
| `firmware_source` | 从 `sample/sle_mesh_test` 源码读取 |
| `derived` | 由真实字段计算得到 |
| `default` | 当前硬件不可直接测量，使用默认值占位 |

## 8. 推荐落盘文件关系

一次正式测试建议生成以下文件：

```text
raw_serial.log
events.jsonl
topology_snapshots.jsonl
routes.json
rounds.jsonl
summary.json
d3qn_state.json
simulation_aligned_metrics.json
simulation_aligned_metrics.jsonl
hardware_test_record.json
report.md
```

其中：

- `summary.json` 和 `rounds.jsonl` 是真实测试主数据。
- `simulation_aligned_metrics.*` 是对齐仿真字段名的数据视图。
- `hardware_test_record.json` 是测试配置、固件参数、广播参数和结果摘要的可追溯留档。
