# 真实测试仿真对齐指标输出格式

本文定义真实 Dijkstra 上位机测试结果如何按 sample 仿真的指标命名导出。目标是让真实硬件结果可以和 `sample` 中 Dijkstra、RPL、D3QN 的 TensorBoard 指标横向对比，同时明确哪些字段来自真实硬件，哪些字段只是默认占位。

## 1. 对齐目标

sample 仿真中的指标分两层产生：

- `environment1.py` 的 `make_step()` 每一步返回 `info` 字典。
- `train_dijkstra.py`、`train_rpl.py`、`train_D3QN_MPNN.py` 将多步指标求均值后写入 TensorBoard scalar。

真实上位机需要额外导出一份仿真同名指标文件，建议文件名：

```text
simulation_aligned_metrics.json
simulation_aligned_metrics.jsonl
```

其中：

- `simulation_aligned_metrics.json` 保存一次 benchmark 的总体聚合结果。
- `simulation_aligned_metrics.jsonl` 保存每个目标节点或每轮测试的对齐记录，便于后续训练或画图。

## 2. 字段命名

仿真训练脚本中使用的主要 TensorBoard scalar 名称如下：

| 仿真字段 | 含义 | 真实测试映射 |
|---|---|---|
| `packet_loss_rate` | 丢包率 | `timeout_count / sent_count`，来自真实 ACK |
| `total_delay` | 总延时 | 成功 ACK 的平均端到端延时，单位秒 |
| `link_utilization` | 路径链路利用率 | 当前硬件不可直接测，默认 0 |
| `global_link_utilization` | 全局链路利用率 | 当前硬件不可直接测，默认 0 |
| `max_link_utilization` | 最大链路利用率 | 当前硬件不可直接测，默认 0 |
| `congested_link_ratio` | 拥塞链路比例 | 当前硬件不可直接测，默认 0 |
| `propagation_delay` | 传播延时 | 当前硬件不可拆分，默认 0 |
| `transmission_delay` | 传输延时 | 当前硬件不可拆分，默认 0 |
| `queueing_delay` | 排队延时 | 当前硬件不可拆分，默认 0 |
| `decomposed_total_delay` | 分解延时之和 | 默认等于真实 `total_delay`，来源标记为 derived |
| `route_length_hops` | 路径跳数 | `len(route) - 1` |
| `mean_delay_per_hop` | 平均每跳延时 | `total_delay / route_length_hops` |
| `route_distance` | 路径距离代理值 | 当前无真实距离，默认 0 |
| `train_score` | 训练综合分 | 可按仿真公式 `total_delay + 10 * packet_loss_rate` 派生 |

注意：仿真中的 `packet_loss_rate` 是根据路径边上的仿真 `packet_loss` 组合出来的；真实测试中的 `packet_loss_rate` 是由 `SEND` 后是否收到 `ACK <dst> <seq>` 判定出来的，两者字段同名但来源不同。

## 4. JSONL 格式

`simulation_aligned_metrics.jsonl` 每行一条记录，建议按 `target` 聚合，也可以按每轮 `round` 输出。

按目标节点聚合示例：

```json
{"schema":"simulation_aligned_metrics.v1","scope":"target","target":"01","packet_loss_rate":0.45,"total_delay":0.2741,"link_utilization":0.0,"global_link_utilization":0.0,"max_link_utilization":0.0,"congested_link_ratio":0.0,"propagation_delay":0.0,"transmission_delay":0.0,"queueing_delay":0.0,"decomposed_total_delay":0.2741,"route_length_hops":1,"mean_delay_per_hop":0.2741,"route_distance":0.0,"train_score":4.7741,"sources":{"packet_loss_rate":"real_ack","total_delay":"real_ack_latency","link_utilization":"default","decomposed_total_delay":"derived","route_length_hops":"derived"}}
```

按每轮输出示例：

```json
{"schema":"simulation_aligned_metrics.v1","scope":"round","target":"03","round":7,"success":true,"packet_loss_rate":0.0,"total_delay":0.4876,"route_length_hops":3,"mean_delay_per_hop":0.1625,"train_score":0.4876,"sources":{"packet_loss_rate":"real_ack","total_delay":"real_ack_latency","route_length_hops":"derived"}}
```

失败轮次示例：

```json
{"schema":"simulation_aligned_metrics.v1","scope":"round","target":"03","round":8,"success":false,"packet_loss_rate":1.0,"total_delay":2.0,"route_length_hops":3,"mean_delay_per_hop":0.6667,"train_score":12.0,"sources":{"packet_loss_rate":"real_ack","total_delay":"ack_timeout","route_length_hops":"derived"}}
```

失败轮次的 `total_delay` 建议填 `ack_timeout`，并将来源标记为 `ack_timeout`；同时在正式统计平均延时时只统计成功 ACK 延时，避免 timeout 拉高成功链路延时。

## 5. 和现有真实日志的关系

现有真实 Dijkstra CLI 已输出：

| 文件 | 用途 |
|---|---|
| `rounds.jsonl` | 每一轮 `SEND`、路径、ACK、timeout、延时 |
| `summary.json` | 总体和分目标丢包率、延时统计 |
| `routes.json` | Dijkstra 路由表 |
| `d3qn_state.json` | D3QN 预留状态，包括 RSSI 边和默认仿真参数 |

新增仿真对齐输出不替代这些文件，而是从它们派生一份同名指标视图，方便后续对接 TensorBoard、D3QN 训练脚本或结果图表。

## 6. TensorBoard 对齐

如果后续要把真实测试也写入 TensorBoard，建议使用同样 tag：

```python
writer.add_scalar("packet_loss_rate", packet_loss_rate, run_index)
writer.add_scalar("total_delay", total_delay, run_index)
writer.add_scalar("link_utilization", link_utilization, run_index)
writer.add_scalar("propagation_delay", propagation_delay, run_index)
writer.add_scalar("transmission_delay", transmission_delay, run_index)
writer.add_scalar("queueing_delay", queueing_delay, run_index)
writer.add_scalar("decomposed_total_delay", decomposed_total_delay, run_index)
writer.add_scalar("route_length_hops", route_length_hops, run_index)
writer.add_scalar("mean_delay_per_hop", mean_delay_per_hop, run_index)
writer.add_scalar("route_distance", route_distance, run_index)
```

真实测试 TensorBoard 只能作为展示和对比，不应把默认字段误认为真实物理测量值。
