# D3QN_MPNN 推理与训练流程

## 1. 拓扑采集（每次测试开始一次，约 17s）

```
发 5 次 RSSI_REQ，间隔 3s
  → 各节点广播自己听到的邻居 RSSI
  → Topology 建无向图（A 听到 B → 双向边，弱信号给高权重，不过滤）
  → rssi_to_weight: ≥-55→1, ≥-65→3, ≥-75→6, ≥-85→12, ≥-95→24, ≥-105→48, else→96
```

11 节点 0x00-0x0A，网关 0x00。节点 8 硬件故障，节点 4 仅被 0x0A 微弱听到（-99~-107dBm）。

---

## 2. 每轮推理决策

### 2.1 总体时序

```
per-round decide(topology, src, dst, demand)
  ├─ [拓扑变时] _refresh_topo_cache()        ← 只在拓扑边集/权重变化时运行
  │     ├─ _edge_betweenness_from_graph       O(N²·K·E log E)  ← 最重
  │     ├─ _edge_feature_array (E,3 numpy)
  │     └─ _first_second_indices              O(E²) MPNN 邻接表
  │
  ├─ k_candidate_paths(graph, src, dst, k=4)  O(K·E log E) ← 每轮仅此
  ├─ _route_feature_array (onehot demand/src/dst)
  ├─ _candidate_edge_indices (路径→边索引)
  │
  └─ DuelingMPNN forward (no_grad)            ~15-20ms
        MPNN T=3 消息传递 (edge_features + first/second)
        → Dueling head (Value + Advantage)
        → Q[k_paths] argmax → selected_action → selected_path
```

### 2.2 缓存策略

拓扑键 = `frozenset((edge.src, edge.dst, edge.weight) for edge in topology.edges)` 

- **缓存命中**（拓扑未变）：跳过 betweenness、edge_features、first/second 全部重建，仅跑 k_candidate_paths + 网络前向，约 **15-20ms**
- **缓存失效**（边权重变化 / 重新采集）：运行一次完整 `_refresh_topo_cache()`，约 80-100ms，之后再次命中缓存

### 2.3 推理指标字段

| 字段 | 含义 |
|------|------|
| `inference_ms` | 模型决策耗时（含摊销在线学习，若本轮触发更新） |
| `point_to_point_ms` | SEND→ACK 端到端延时（单次 SEND 模式） |
| `total_latency_ms` | `inference_ms + point_to_point_ms` |
| `online_learn_ms` | 本轮触发学习更新耗时（单独记录，已计入 inference_ms） |

## 3. 在线学习（OnlineD3QNLearner）

- **触发**：每 `interval`（100）轮触发一次梯度更新，`epochs=1`
- **奖励**：`r = 1 - min(latency_ms / latency_norm, 1.0)`，失败 `r = -1.0`
- **更新方式**：方向性 bandit —— `target = q_sel.detach() + nudge × r`，smooth_l1_loss
- **摊销耗时**：约 9ms/轮（总学习时间 / 总轮次）
- **存储**：`online_latest.pt` / `online_best.pt` 在测试日志目录，**永不覆盖** `checkpoints/` 中的 `latest.pt` / `best.pt`

---

## 4. 离线训练（run_sample_training）

```
仿真环境：sample/D3QN_MPNN_TimePER/
  → DuelingMPNN + PER 经验回放
  → 每 20 轮评估 + 保存 checkpoint
  → 支持 --resume 续训
```

### Checkpoint 层级

| 文件 | 说明 |
|------|------|
| `checkpoints/d3qn_mpnn/overnight_base.pt` | 测试基线（iter=260, eval=+0.0828，最优） |
| `checkpoints/d3qn_mpnn/best.pt` | 全量训练最佳（iter=1120, eval=+0.075） |
| `/tmp/overnight/d3qn_overnight/latest.pt` | 通宵训练当前进度 |

### 通宵编排（orchestrate.sh）

```
无限循环：
  5 次在线学习测试（各 3 轮 × N 对 ≈ ~270 轮）
    参数从 /tmp/overnight/params.txt 读取：INT LR NUDGE EP
  → 续训 500 轮（仿真）
  → eval(新) > eval(overnight_base) → 提升测试基线
  单实例锁（flock），dongle 失联自动等待重连
```

## 5. 推理时间对比

| 阶段 | 第40次（无学习、无缓存） | 之前（有学习、无缓存） | 优化后（有缓存、有学习） |
|------|------|------|------|
