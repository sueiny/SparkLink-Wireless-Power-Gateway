# 手动测试指令集合

所有命令在 `app/` 目录下执行，串口 `/dev/ttyUSB0`，dongle 地址 `16`（0x10）。

---

## 一、Sample Dijkstra（纯净最小跳数）

### 基准测试 bench

```bash
# 标准测试（5轮，ACK timeout 3s）
.venv-d3qn/bin/python -m pc_dijkstra_cli bench \
  --port /dev/ttyUSB0 --dongle-addr 16 --rounds 5 \
  --send-mode single_send --no-retry --ack-timeout 3.0 \
  --hop-penalty 0 --interval 2.0 --route-mode sample_dijkstra

# 快速测试（2轮，ACK timeout 2s）
.venv-d3qn/bin/python -m pc_dijkstra_cli bench \
  --port /dev/ttyUSB0 --dongle-addr 16 --rounds 2 \
  --send-mode single_send --no-retry --ack-timeout 2.0 \
  --hop-penalty 0 --interval 2.0 --route-mode sample_dijkstra

# 带暂停功能（s=暂停，y=重新采集RSSI并继续）
.venv-d3qn/bin/python -m pc_dijkstra_cli bench \
  --port /dev/ttyUSB0 --dongle-addr 16 --rounds 5 \
  --send-mode single_send --no-retry --ack-timeout 5.0 \
  --hop-penalty 0 --interval 2.0 --route-mode sample_dijkstra --enable-pause
```

### 参数说明

| 参数 | 值 | 说明 |
|------|-----|------|
| `--route-mode` | sample_dijkstra | 纯净最小跳数，边权恒定 0.005 |
| `--hop-penalty` | 0 | 无跳数惩罚（sample 模式强制为 0） |
| `--send-mode` | single_send | 单次发送：网关→目标 |
| `--no-retry` | - | 禁用 ACK 超时重试 |
| `--ack-timeout` | 5.0 | ACK 超时秒数（4s=12%丢包，5s=7%） |
| `--interval` | 2.0 | 发包间隔秒数 |
| `--enable-pause` | - | 启用暂停功能（s暂停，y上报） |

### 暂停功能

| 按键 | 功能 |
|------|------|
| `s` + 回车 | 暂停（当前 round 完成后生效） |
| `y` + 回车 | 重新采集 RSSI 并继续测试（最多重试 3 次） |

### 其他命令

```bash
# 监听串口
.venv-d3qn/bin/python -m pc_dijkstra_cli listen --port /dev/ttyUSB0

# 查看路由
.venv-d3qn/bin/python -m pc_dijkstra_cli routes --port /dev/ttyUSB0

# 发送单包
.venv-d3qn/bin/python -m pc_dijkstra_cli send --port /dev/ttyUSB0 --dst 5 --payload AABBCC
```

## 二、D3QN

### 基准测试 bench

```bash
# 标准测试
.venv-d3qn/bin/python -m pc_d3qn_cli bench \
  --port /dev/ttyUSB0 --dongle-addr 16 --rounds 5 \
  --send-mode single_send --no-retry --ack-timeout 5.0 \
  --interval 2.0

# 指定 checkpoint
.venv-d3qn/bin/python -m pc_d3qn_cli bench \
  --port /dev/ttyUSB0 --dongle-addr 16 --rounds 5 \
  --send-mode single_send --checkpoint checkpoints/d3qn_mpnn/latest.pt
```

### 在线学习 bench（测试期边测边学，每 N 轮微调）

> 每 `--online-interval` 个传输轮，用真实 ACK 经验微调模型并存**独立** checkpoint，
> 学习耗时计入推理时间。基线默认自动用 **best.pt**（不覆盖 latest.pt / best.pt）。
> 注意：d3qn 的 bench **没有** `--no-retry`（那是 dijkstra 的）。

```bash
# 开启在线学习（默认每 50 轮、以 best.pt 为基线）
.venv-d3qn/bin/python -m pc_d3qn_cli bench \
  --port /dev/ttyUSB0 --dongle-addr 16 --rounds 2 \
  --send-mode single_send --rssi-requests 10 --ack-timeout 5.0 --interval 2.0 \
  --online-learn

# 更频繁更新 + 学习更激进（每 30 轮）
.venv-d3qn/bin/python -m pc_d3qn_cli bench \
  --port /dev/ttyUSB0 --dongle-addr 16 --rounds 3 \
  --send-mode single_send --rssi-requests 10 --ack-timeout 5.0 --interval 2.0 \
  --online-learn --online-interval 30 --online-lr 3e-4 --online-nudge 0.8
```

#### 在线学习参数

| 参数 | 默认 | 说明 |
|------|------|------|
| `--online-learn` | 关 | 开启测试期在线学习（bandit 单步、延迟塑形奖励、贪心不探索） |
| `--online-interval` | 50 | 每多少传输轮触发一次更新 |
| `--online-lr` | 1e-4 | 学习率（小步，避免破坏预训练成果） |
| `--online-epochs` | 2 | 每次更新在经验窗口上的训练轮数 |
| `--online-nudge` | 0.5 | 方向性步长：目标 = 当前Q + 步长×奖励（失败下压、成功上推） |

#### 查看学习效果

```bash
LATEST=$(ls -t logs/d3qn_hw/ | head -1)
# 每次在线更新事件（损失 / 窗口奖励 / 更新耗时 / 是否提升 best）
grep d3qn_online_update "logs/d3qn_hw/$LATEST/原始JSON数据/events.jsonl"
# 独立 checkpoint（绝不动 latest.pt / best.pt）
ls -la "logs/d3qn_hw/$LATEST/"online_*.pt
```

> 节点移动场景：测试中途物理移动某节点，原路径超时会被学着降权，后续自动改选更优候选。

### 训练

```bash
# 恢复训练

```

## 四、日志目录

| 算法 | 默认日志路径 |
|------|-------------|
| Dijkstra | `logs/dijkstra_hw/第N次测试/` |
| D3QN | `logs/d3qn_hw/第N次测试/` |

每次测试自动递增序号。日志内含：

- `测试结果汇报.md`：汇总报告
- `原始JSON数据/summary.json`：结构化数据
- `原始JSON数据/rounds.jsonl`：逐轮数据
- `原始JSON数据/state.json`：RSSI 拓扑快照
- `拓扑图.svg`：拓扑可视化

## 五、已知问题

- **节点 4 单向可达**：只有网关→4 的入边（RSSI -84），无其他节点→4 的边。物理位置问题。
- **Dongle 偶尔断开**：需物理拔插 USB。代码会自动保存已有结果。
- **RSSI 采集参数**：已硬编码 rssi_seconds=20, rssi_requests=10（CLI 参数已移除）。
