## 二、Dijkstra 测试

### 2.1 基本测试命令

```bash
cd /home/sueiny/rk3506_linux6.1_v1.2.0/app/广播组网上位机/app

# 最优参数组合（当前最佳）
python -m pc_dijkstra_cli bench \
  --port /dev/ttyUSB0 \
  --dongle-addr 16 \
  --rounds 2 \
  --send-mode single_send \
  --rssi-seconds 20 \
  --rssi-requests 10 \
  --max-hops 3 \
  --no-retry \
  --ack-timeout 5.0 \
  --hop-penalty 8 \
  --interval 2.0
```

### 2.2 参数说明

| 参数 | 当前最优值 | 说明 |
|------|-----------|------|
| `--port` | /dev/ttyUSB0 | 串口路径 |
| `--dongle-addr` | 16 | dongle 地址，排除于路由 |
| `--rounds` | 2 | 每对节点测试轮次 |
| `--send-mode` | single_send | 单次发送模式 |
| `--rssi-seconds` | 20 | RSSI 采集时长（秒） |
| `--rssi-requests` | 10 | RSSI 采集请求次数 |
| `--max-hops` | 3 | 最大跳数 |
| `--no-retry` | (flag) | 禁用 ACK 超时重传 |
| `--ack-timeout` | 5.0 | ACK 超时时间（秒） |
| `--hop-penalty` | 8 | 每跳惩罚（0=禁用） |
| `--interval` | 2.0 | 发包间隔（秒） |
| `--enable-congestion` | (flag) | 启用节点拥塞惩罚 |
| `--bidirectional` | (flag) | 仅使用双向边（实验性） |

### 2.3 其他有用命令

```bash
# 查看路由表
python -m pc_dijkstra_cli routes --port /dev/ttyUSB0

# 监听串口数据
python -m pc_dijkstra_cli listen --port /dev/ttyUSB0

# 发送单包测试
python -m pc_dijkstra_cli send --port /dev/ttyUSB0 --dst 5 --payload AABBCC

# 间隔扫描
python -m pc_dijkstra_cli sweep --port /dev/ttyUSB0 --rounds 10 --intervals "0.5,1.0,1.5,2.0"
```

## 三、D3QN 测试

### 3.1 训练

```bash

python -m pc_d3qn_cli train --iterations 200 --training-episodes 20

# 恢复训练
python -m pc_d3qn_cli train --resume checkpoints/d3qn_mpnn/latest.pt --iterations 200

  --port /dev/ttyUSB0 \

```

## 四、日志结构

每次测试自动在 `logs/` 下创建带序号的目录：

```
logs/dijkstra_hw/第N次测试/
├── 测试结果汇报.md          # Markdown 报告
├── 测试指标汇总.xlsx        # Excel 汇总
├── 拓扑图.svg               # 拓扑可视化
├── 拓扑图.txt               # 拓扑文本
├── 原始串口日志.log          # 完整串口通信记录
└── 原始JSON数据/
    ├── summary.json          # 汇总统计
    ├── rounds.jsonl          # 逐轮详细数据（每行一个 JSON）
    ├── state.json            # RSSI 拓扑快照
    ├── routes.json           # 路由表
    ├── events.jsonl          # 事件日志
    └── simulation_aligned_metrics.jsonl
```
