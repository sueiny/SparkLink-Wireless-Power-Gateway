# 代码结构详解

## 一、Dijkstra 核心文件

### 1.1 routing.py（路由算法）

```python
# 核心函数
rssi_to_weight(rssi)          # RSSI → 权重映射
rssi_to_reliable_weight(rssi) # reliable 模式权重
dijkstra(graph, src, dst, max_hops, hop_penalty, congestion_penalty)
build_route_table(graph, nodes, max_hops, gateway, hop_penalty, congestion_penalty)
_calculate_node_congestion_penalty(graph, gateway, penalty_factor)
update_congestion_penalty(congestion_penalty, high_latency_nodes, penalty_increase)
```

**dijkstra() 函数**：
- 输入：graph（邻接表）、src、dst、max_hops、hop_penalty、congestion_penalty
- 输出：Route(src, dst, path, cost, status)
- 权重计算：`new_cost = current_cost + edge_weight + hop_penalty + node_penalty`
- 最大跳数限制：`max_hops`（当前设为 3）

### 1.2 topology.py（拓扑管理）

```python
class Topology:
    def update_from_rssi_report(report)  # 从 RSSI 报告更新边
    def graph(route_mode, min_rssi, bidirectional)  # 构建邻接表
    def route(src, dst, ...)  # 计算单条路由
    def routes(route_mode, ...)  # 计算所有路由
```

**graph() 方法**：
- 过滤 stale 边（`stale_seconds`）
- 过滤 `edge.dst == gateway` 的边
- 过滤 dongle 地址的边
- 过滤 RSSI < min_rssi 的边
- `bidirectional=True` 时只保留双向边

### 1.3 benchmark.py（测试核心）

```python
# 关键函数
run_benchmark(...)                    # 主测试函数
_resolve_healthy_matrix_routes(...)   # 计算完整路由（网关→源→目标）
_select_dijkstra_pair_route(...)      # 选择点对点路径（带历史惩罚）
_send_and_wait_ack_with_retry(...)    # 发送并等待 ACK（带重试）
_send_and_wait_ack(...)               单次发送并等待 ACK
_candidate_paths(...)                 # K-shortest 路径枚举
_has_ack_return_path(...)             # ACK 回程路径检查
_write_routes_json(...)               # 写入路由表 JSON
```

**run_benchmark() 流程**：
1. 初始化串口（含 dongle 等待）
2. RSSI 采集（`rssi-seconds` × `rssi-requests`）
3. 验证路由表完整性
4. 对每对节点（source, target）：
   a. 计算路由（`_resolve_healthy_matrix_routes`）
   b. 发送 SEND 命令
   c. 等待 ACK
   d. 记录结果
5. 生成报告

### 1.4 serial_client.py（串口通信）

### 1.5 protocol.py（协议）


### 1.6 main.py（CLI）

## 二、D3QN 核心文件

### 2.1 model.py（D3QN 模型）

- Dueling Double DQN + MPNN
- 输入：状态向量（拓扑 + RSSI + 节点信息）
- 输出：每个候选路径的 Q 值

### 2.2 state.py（状态构建）

```python
build_d3qn_state(topology, src, dst, ...)  # 构建 D3QN 输入状态
k_candidate_paths(graph, src, dst, k, ...) # 生成 K 候选路径
```

### 2.3 train_runner.py（训练）

- 支持自动恢复训练
- checkpoint 保存在 `checkpoints/d3qn_mpnn/`
- `latest.pt` 是最新 checkpoint

### 2.4 benchmark.py（D3QN 测试）

- 与 Dijkstra 类似，但路由计算使用 D3QN 模型
- 推理时间 ~20ms（vs Dijkstra ~0.2ms）

## 三、关键数据结构

### 3.1 RoundResult（单轮结果）

```python
@dataclass
class RoundResult:
    target: int               # 目标节点
    round_index: int          # 轮次
    route: list[int]          # 路由路径
    cost: float               # 路由成本
    success: bool             # 是否成功
    latency_ms: float         # 延迟（ms）
    source_to_target_ms: float
    total_latency_ms: float
    inference_ms: float
    status: str               # success/timeout/unreachable
```

### 3.2 Edge（拓扑边）

```python
@dataclass
class Edge:
    src: int
    dst: int
    rssi: int
    weight: int
    updated_at: float
    direction: str
    source: str
    samples: list[int]
```

### 3.3 Route（路由）

```python
@dataclass
class Route:
    src: int
    dst: int
    path: list[int]
    cost: float
    status: str  # valid/unreachable
```

```
      → _resolve_healthy_matrix_routes(
          topology, gateway, source, target,
          route_mode, path_history,
          max_hops, hop_penalty, congestion_penalty, bidirectional
        )
        → _select_dijkstra_pair_route(...)
          → topology.route(...)
            → dijkstra(graph, src, dst, max_hops, hop_penalty, congestion_penalty)
```
