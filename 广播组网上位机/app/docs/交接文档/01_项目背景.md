# 项目背景

## 一、项目概述

本项目是一个基于 Mesh 自组网的上位机测试系统，用于对比两种路由算法在真实硬件上的性能：

1. **Dijkstra**：经典最短路径算法，基于 RSSI 信号强度计算权重
2. **D3QN_MPNN**：深度强化学习路由算法，基于 Dueling Double DQN + Message Passing Neural Network

## 二、硬件环境

### 2.1 网络拓扑

- **节点数量**：11 个（编号 0x00 - 0x0A）
- **网关节点**：0x00（dongle，地址 0x10 = 十进制 16）
- **通信方式**：SLE（类似 BLE）无线 Mesh 网络
- **串口**：`/dev/ttyUSB0`，波特率 115200
- **dongle 芯片**：CH340 USB 转串口

### 2.2 Dongle 关键信息

- dongle 地址：0x10（16 十进制），测试时用 `--dongle-addr 16` 排除
- dongle 每次连接时需要等待 2 秒初始化（代码已处理）
- dongle 偶尔会死机，需要物理拔插 USB 重置
- dongle 启动时会输出 boot 日志（Flash Init、RF cali 等）

### 2.3 协议格式

- **SEND 命令**：`SEND <目标十进制> <跳数> <路径十六进制...> <载荷>\r\n`
  - 示例：`SEND 5 3 00 04 05 aabbcc\r\n`（目标=5，3跳，路径=0→4→5）
- **RSSI_REQ**：`RSSI_REQ\r\n`（请求所有节点报告邻居 RSSI）
- **RSSI_REPORT**：`RSSI_REPORT src=X count=N [邻居:RSSI] ...`
- **ACK**：目标节点收到数据后回复 ACK（二进制帧）

## 四、RSSI 拓扑采集原理

1. 上位机发送 `RSSI_REQ` 命令
2. 所有节点广播自己的邻居 RSSI 报告
3. 上位机收集报告，构建有向图：`src → dst, weight = rssi_to_weight(rssi)`
4. RSSI 权重映射（当前版本）：
   - RSSI ≥ -55: weight=4
   - RSSI ≥ -65: weight=5
   - RSSI ≥ -75: weight=6
   - RSSI ≥ -85: weight=12
   - < -85: 不可用，边被丢弃
5. 拓扑不对称：A→B 有边不代表 B→A 有边
6. 每次采集结果可能不同（无线信道波动）

## 五、已知硬件问题

1. **节点 4 无入边**：除网关外，其他节点到节点 4 没有 RSSI 边，导致节点 4 作为目标时经常不可达
2. **拓扑不对称**：网关有 6-8 条出边，但 0 条入边（设计如此，网关是源节点）
3. **dongle 死机**：偶尔需要物理拔插，dmesg 会显示 USB disconnect
4. **RSSI 波动**：每次采集得到的边集不同，导致测试结果不稳定
